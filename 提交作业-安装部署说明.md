仓库管理系统 - 安装部署说明

一、项目概述

项目名称：仓库管理系统 (Warehouse Management System)
技术架构：SpringBoot + Vue 前后端分离
部署方式：本地部署（源码方式）

---

二、系统要求

2.1 硬件要求
CPU: 2核及以上
内存: 4GB及以上
硬盘: 至少10GB可用空间

2.2 软件环境
操作系统：Windows 10/11、macOS 或 Linux
JDK：1.8 或以上版本
Maven：3.6 或以上版本
MySQL：8.0 或以上版本
Node.js：14.x 或以上版本
npm：6.x 或以上版本

---

三、快速安装指南

3.1 数据库安装与配置

3.1.1 创建数据库
登录MySQL
mysql -u root -p

创建数据库
CREATE DATABASE warehouse CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

3.1.2 执行初始化脚本
进入SQL脚本目录
cd sql

按顺序执行以下SQL文件
mysql -u root -p warehouse < init.sql
mysql -u root -p warehouse < add_barcode.sql
mysql -u root -p warehouse < create_alert_table.sql
mysql -u root -p warehouse < optimize_storage_names.sql

3.1.3 修改数据库配置（可选）
如需修改数据库连接信息，请编辑：
backend/src/main/resources/application.yml

修改以下配置：
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/warehouse?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=GMT%2b8&allowPublicKeyRetrieval=true
    username: root
    password: 你的密码

---

3.2 后端安装与启动

3.2.1 编译打包
进入后端目录
cd backend

Maven打包
mvn clean package

等待编译完成，将在 target 目录生成 warehouse-0.0.1-SNAPSHOT.jar

3.2.2 启动后端服务
方式一：直接运行jar包
java -jar target/warehouse-0.0.1-SNAPSHOT.jar

方式二：在IDE中运行
找到 backend/src/main/java/com/warehouse/WarehouseApplication.java
右键 -> Run 'WarehouseApplication'

3.2.3 验证后端服务
在浏览器访问
http://localhost:9090

或使用curl测试
curl http://localhost:9090/api/user/login

预期结果：返回JSON响应 {"code": 200, "msg": "success"}

---

3.3 前端安装与启动

3.3.1 安装依赖
进入前端目录
cd frontend

安装依赖（首次运行需要较长时间）
npm install

3.3.2 配置后端接口地址
编辑 frontend/src/api/index.js，确保API基础地址正确：
const BASE_URL = 'http://localhost:9090'

3.3.3 启动前端服务
开发模式启动
npm run serve

3.3.4 访问系统
在浏览器中打开
http://localhost:8080

五、生产环境部署（可选）

5.1 前端构建
cd frontend
npm run build

构建产物将生成在 frontend/dist 目录，可部署到Nginx或Apache服务器。

5.2 使用Docker部署（推荐）

创建后端Dockerfile
FROM openjdk:8-jdk-alpine
VOLUME /tmp
COPY target/warehouse-0.0.1-SNAPSHOT.jar app.jar
ENTRYPOINT ["java","-jar","/app.jar"]

构建并运行
构建镜像
docker build -t warehouse-backend .

运行容器
docker run -d -p 9090:9090 warehouse-backend

六、常见问题排查

6.1 后端启动失败
问题：端口9090被占用
  解决：修改 application.yml 中的 server.port 配置

问题：数据库连接失败
  解决：检查MySQL服务是否启动，用户名密码是否正确

6.2 前端启动失败
问题：npm install 报错
  解决：删除 node_modules 和 package-lock.json，重新运行 npm install

问题：跨域错误
  解决：确保后端配置了CORS，检查 WebMvcConfig.java

6.3 功能异常
问题：数据无法保存
  解决：检查数据库表是否创建成功，查看后端日志
